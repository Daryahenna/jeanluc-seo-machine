## 2. Frontmatter — копируй и заполняй

```yaml
---
title: "«Запрос клиента» — короткий рабочий ответ"
description: "1-2 предложения с обещанием результата + ключевые слова. Макс 160 символов."
promise: "Через X минут у тебя будет [конкретный результат] — без [главного страха клиента]."
category: "Возражения"
chips:
  - "Возражения"
  - "Скрипт внутри"
  - "X мин чтения"
hero_emoji: "💸"
hero_caption: "Короткий подзаголовок-приманка под эмодзи"
author: "Jean-Luc Médéric"
date: 2026-MM-DD
reading_time: "X мин чтения"
lang: ru
slug: short-url-slug-translit
faq:
  - q: "Первый частый вопрос?"
    a: "Прямой ответ 2-4 предложения."
  - q: "Второй?"
    a: "Ответ."
  - q: "Третий?"
    a: "Ответ."
  - q: "Четвёртый?"
    a: "Ответ."
---
```

Правила:
- `title` — естественный поисковый запрос, 40-60 символов, в ёлочках
- `description` — 140-160 символов, с ключом + обещанием
- `promise` — формат «Через X минут у тебя будет…»
- `slug` — короткий, латиницей через дефис, без `kak`/`chto`
- `faq` — ровно 4 вопроса. Они отрендерятся в конце как accordion. **Не дублируй FAQ в теле статьи**
- `chips` — 3 шт: категория, фишка статьи, время чтения
- `hero_emoji` — один тематический эмодзи. Палитра: 💸 деньги, ⏱️ время, 🎯 точность, 🤝 переговоры, 🔥 срочность, 🧠 мышление, 📞 звонки, ✉️ переписка

## 3. Структура статьи (порядок секций)

1. **Зацепляющий intro** — 3-4 коротких абзаца. Личная история, цифра, провокация. **Никаких TL;DR.**
2. **Главная мысль** в `<div class="callout is-key">` с `<p class="callout-title">⚡️ Главная мысль</p>`
3. **H2 «Почему [боль]»** — раскрываем суть проблемы
4. **`<ul class="check">`** — чек-лист причин/факторов, 5 пунктов
5. **`<div class="callout is-tip">`** с правилом: `💡 Правило №1`
6. **H2 «Что НЕ надо делать»** — 3 анти-примера в `<div class="bad-example">`. После каждого — короткий разбор «что не так»
7. **`<div class="callout is-quote">`** — цитата Michael Bang или Jean-Luc + `<cite>` с источником
8. **H2 «Шаг 1»**, «Шаг 2», «Шаг 3» — конкретный процесс. После каждого — `<div class="phrase-card">` с готовой фразой
9. **H2 со сценариями** через H3 (A, B, C, D) — каждый со скриптом в `<div class="good-example">`
10. **Inline-баннер:** `{% include paris-banner.html utm_medium="guide_inline" %}` — одна строка, ничего больше
11. **H2 «Реальный диалог»** — диалог-пьеса в `<div class="dialog">`
12. **H2 «N готовых фраз»** — 8 шт в `<div class="phrase-card">`, по одной фразе в карточке
13. **H2 «Чек-лист перед…»** — `<ul class="check">` 5 вопросов себе
14. **H2 «Частые ошибки»** — `<div class="callout is-warn">` с пронумерованным списком
15. **H2 «Главный вывод»** — мощная финальная мысль, 1-2 абзаца

**ВАЖНО:** в теле статьи **НЕ пиши** outro («Хочешь натренировать…») и `## FAQ`. Они автоматически в layout из frontmatter `faq:` и общего outro-шаблона.

## 4. Голос

Личный, на «ты», как тренер-ровесник. От первого лица.

**Хорошо:**
- «У меня в первый год было 8 из 10. У сильных продавцов — 2-3.»
- «Звучит банально, но 80% продавцов не делают этого.»

**Плохо:**
- «Уважаемый читатель, в данной статье мы рассмотрим…»
- «Согласно методологии Golden Key of Sales…» (звучит как реферат)

**Длина абзацев:** 1-3 коротких предложения максимум.

**Цифры конкретные:** «8 из 10», «90 дней», «€18,000», «30-40%». Не «много», «часто», «обычно».

**Сторителлинг:** хотя бы один реальный диалог-пьеса в каждой статье.

**В диалогах:** «Менеджер» и «Клиент». **НЕ «Жан-Люк»** — это деперсонализирует и работает на любого продавца.

## 5. Длина

- **Целевая:** 1800-2500 слов = ~7 минут чтения
- **Минимум:** 1500 слов
- **Максимум:** 3500 слов для pillar-страниц

## 6. Дизайн-система — готовые HTML-классы

Все классы в `assets/css/main.css`. Не изобретай новые.

| Класс | Когда |
|---|---|
| `<div class="callout is-key">` | Главная мысль (1 раз) |
| `<div class="callout is-tip">` | Лайфхак, правило, подсказка |
| `<div class="callout is-warn">` | Список ошибок, предупреждение |
| `<div class="callout is-quote">` | Цитата эксперта с `<cite>` |
| `<div class="bad-example">` | Анти-пример |
| `<div class="good-example">` | Готовый скрипт |
| `<div class="phrase-card">` | Готовая фраза для копипаста |
| `<div class="dialog">` + `<p class="dialog-line">` | Диалог-пьеса. `is-them` = клиент |
| `<ul class="check">` | Чек-лист |
| `{% include paris-banner.html utm_medium="guide_inline" %}` | Inline CTA |

**В dialog-line:** `<span class="who">Менеджер:</span><span class="what">текст</span>`

**В callout заголовок:** первым `<p class="callout-title">⚡️ Заголовок</p>`

**ANTI-PATTERN:** не разрывай `<a>` через несколько строк markdown с блочными элементами внутри (`<h3>` внутри `<a>`). Kramdown сломает — `</a>` вылезет как текст. Используй `{% include %}`.

## 7. CTA-стратегия

В одной статье **ровно 3 CTA**:
1. **Top banner** — автоматически в layout
2. **Inline banner** — ты ставишь в середине после блока сценариев A/B/C/D
3. **Outro + bottom banner** — оба автоматически в layout

Больше CTA не лепи. Свои блоки «купи курс» в теле — нет.

## 9. Workflow публикации (как я пушу сама)

**НЕ** работаю напрямую через `D:\Desktop\ЖАН ЛЮК\сео-машина` для коммитов — там GitHub Desktop держит lock. Использую clone-cycle:

```bash
# 1. Clone (или git pull если уже клонировано)
rm -rf /tmp/jlm && git clone https://github.com/Daryahenna/Jean-Luc-Mederic.git /tmp/jlm
# Credentials из ~/.git-credentials (token уже настроен)

# 2. Редактирую файлы в /tmp/jlm/

# 3. Параллельно копирую в mount, чтобы Дашин GitHub Desktop их увидел
cp /tmp/jlm/_guides/ru/<slug>.md "/sessions/.../mnt/сео-машина/_guides/ru/<slug>.md"

# 4. Commit + push
cd /tmp/jlm && git add -A && \
  git commit -m "Add guide: <slug> — <short>" && \
  git push origin main
```

После push — Даша через 1-2 мин делает Ctrl+Shift+R на странице статьи. В GitHub Desktop делает Fetch + Pull для синхронизации.

## 12. Anti-patterns — никогда

1. ❌ TL;DR / «Короткий ответ» сухим списком в начале — выглядит как реферат
2. ❌ Обращение на «вы» — теряется доверительность
3. ❌ Заголовки «5 советов как…» — кликбейт, низкое доверие
4. ❌ `==выделение==` (highlight syntax) — Kramdown не парсит
5. ❌ `<a>` обёрнутый вокруг блочных элементов через переносы строк — сломается
6. ❌ Дублирование FAQ или outro в теле (они в layout)
7. ❌ «По данным исследований…» без ссылки — звучит как ложь
8. ❌ Эмодзи в потоке текста (только в callout titles и hero)
9. ❌ Канцелярит: «является», «осуществляет», «представляет собой»
10. ❌ Длинные предложения с придаточными — режь короче
11. ❌ «Жан-Люк» в диалогах — пиши «Менеджер»
12. ❌ Свои самописные CTA-блоки — используй inline-баннер из include
